# Техническое задание: Чекбокс "Использовать данные кампании"

## Версия документа
- **Версия:** 1.0

- **Автор:** SMM Manager Team

## 1. Цель проекта
Реализовать функциональность, которая позволяет при генерации контента автоматически подставлять в промпт реальные данные кампании из базы данных Directus.

## 2. Функциональные требования

### 2.1 Пользовательский интерфейс
- **Элемент:** Чекбокс в диалоге генерации контента
- **Подпись:** "Использовать данные кампании"
- **По умолчанию:** Выключен (unchecked)
- **Расположение:** В форме генерации контента, рядом с другими опциями

### 2.2 Данные кампании для использования

**Источники данных в Directus:**
1. **Таблица `user_campaigns`** - основная информация о кампании
2. **Коллекция `business_questionnaire`** - анкета компании

**Поля для получения:**
- **`link`** (string) - URL сайта кампании из `user_campaigns`
- **`questionnaire`** (text) - данные анкеты из коллекции `business_questionnaire`

**Метод получения данных из Directus:**

**1. Данные кампании:**
- **API endpoint:** `GET /items/user_campaigns/{campaignId}`
- **Авторизация:** Bearer токен пользователя
- **Формат ответа:** 
  ```json
  {
    "data": {
      "id": "campaign-uuid",
      "link": "https://example.com",
      "user_id": "user-uuid"
    }
  }
  ```

**2. Анкета компании:**
- **API endpoint:** `GET /items/business_questionnaire?filter[campaign_id][_eq]={campaignId}`
- **Авторизация:** Bearer токен пользователя (аналогично другим коллекциям системы)
- **Фильтрация:** По `campaign_id` и автоматическая проверка принадлежности пользователю
- **Формат ответа:**
  ```json
  {
    "data": [
      {
        "id": "questionnaire-uuid",
        "campaign_id": "campaign-uuid",
        "company_name": "Название компании",
        "business_description": "Описание бизнеса...",
        "target_audience": "Целевая аудитория...",
        "user_id": "user-uuid"
      }
    ]
  }
  ```

### 2.3 Логика работы

#### При выключенном чекбоксе:
- Система использует только исходный промпт пользователя
- Никаких дополнительных запросов к базе данных не делается

#### При включенном чекбоксе:
1. Система получает ID текущей активной кампании
2. Делает запрос к Directus API: `GET /items/user_campaigns/{campaignId}`
3. Использует токен текущего пользователя для авторизации
4. Получает поля `link` и `questionnaire`
5. **Дополняет исходный промпт:**
   ```
   {исходный_промпт}
   
   Дополнительная информация о компании:
   Сайт компании: {link}
   О компании: {questionnaire}
   ```
6. Отправляет улучшенный промпт в AI сервис

### 3.2 Бэкенд (Node.js/Express)

**Файл:** `server/routes.ts`

**Новый маршрут:** `POST /api/generate-content`

**Логика:**
```typescript
app.post("/api/generate-content", authenticateUser, async (req, res) => {
  const { prompt, keywords, tone, campaignId, platform, service, useCampaignData } = req.body;
  
  let enhancedPrompt = prompt;
  
  if (useCampaignData) {
    try {
      const authHeader = req.headers.authorization || '';
      const token = authHeader.replace('Bearer ', '');
      
      // 1. Получаем данные кампании
      const campaignResponse = await axios.get(
        `${DIRECTUS_URL}/items/user_campaigns/${campaignId}`,
        { headers: { 'Authorization': `Bearer ${token}` } }
      );
      
      const { link } = campaignResponse.data.data;
      
      // 2. Получаем анкету компании
      const questionnaireResponse = await axios.get(
        `${DIRECTUS_URL}/items/business_questionnaire?filter[campaign_id][_eq]=${campaignId}`,
        { headers: { 'Authorization': `Bearer ${token}` } }
      );
      
      const questionnaireData = questionnaireResponse.data.data[0]; // Берем первую анкету
      
      // 3. Дополняем промпт
      if (link || questionnaireData) {
        enhancedPrompt += '\n\nДополнительная информация о компании:';
        if (link) enhancedPrompt += `\nСайт компании: ${link}`;
        if (questionnaireData) {
          if (questionnaireData.company_name) 
            enhancedPrompt += `\nНазвание компании: ${questionnaireData.company_name}`;
          if (questionnaireData.business_description) 
            enhancedPrompt += `\nОписание бизнеса: ${questionnaireData.business_description}`;
          if (questionnaireData.target_audience) 
            enhancedPrompt += `\nЦелевая аудитория: ${questionnaireData.target_audience}`;
        }
      }
    } catch (error) {
      console.error('Ошибка получения данных кампании:', error);
      // Продолжаем с исходным промптом
    }
  }
  
  // Отправляем в AI сервис
  // ...существующая логика генерации
});
```

## 4. Обработка ошибок

### 4.1 Сценарии ошибок:
- **Нет доступа к кампании:** Логировать ошибку, использовать исходный промпт
- **Пустые данные кампании:** Использовать только доступные поля
- **Ошибка сети:** Логировать ошибку, использовать исходный промпт
- **Неверный токен:** Возвращать ошибку авторизации

### 4.2 Логирование:

console.log(`[CAMPAIGN-DATA] Получение данных для кампании ${campaignId}`);

## 5. Безопасность и система авторизации

### 5.1 Использование существующей системы авторизации
**Принцип:** Использовать ту же систему авторизации, что и в разделе "Кампании"

**Схема авторизации:**
1. **Получение токена:** Из `localStorage` или `authStore` (текущая авторизованная сессия)
2. **Формат токена:** Bearer токен Directus API 
3. **Передача токена:** В заголовке `Authorization: Bearer {token}`
4. **Middleware:** Использовать существующий `authenticateUser` middleware

**Аналогия с разделом "Кампании":**
```typescript
// Существующая логика (как в /api/campaigns)
const authHeader = req.headers.authorization || '';
const token = authHeader.replace('Bearer ', '');

// Запрос к Directus с токеном пользователя
const response = await axios.get(`${DIRECTUS_URL}/items/user_campaigns/${campaignId}`, {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
```

### 5.2 Проверка принадлежности кампании
**Автоматическая фильтрация:** Directus автоматически фильтрует данные по `user_id` на основе токена

**Дополнительная проверка:**
- Проверять, что в ответе от Directus есть данные (`response.data.data`)
- При отсутствии данных - кампания не принадлежит пользователю или не существует

### 5.3 Валидация:
- Проверять существование `campaignId` в запросе
- Валидировать формат полученных данных
- Проверять валидность токена авторизации

## 6. Тестирование

### 6.1 Тестовые сценарии:
1. **Чекбокс выключен** - генерация с исходным промптом
2. **Чекбокс включен + данные есть** - генерация с дополненным промптом
3. **Чекбокс включен + данных нет** - генерация с исходным промптом
4. **Ошибка авторизации** - возврат ошибки 401
5. **Кампания не принадлежит пользователю** - возврат ошибки 403

### 6.2 Проверка результата:
- В сгенерированном контенте должна появляться информация из полей кампании
- Логи должны показывать процесс получения данных
- Ошибки должны корректно обрабатываться

- Большой размер анкеты может превысить лимиты AI API

### 8.2 Ограничения:
- Функция доступна только авторизованным пользователям
- Работает только с кампаниями, принадлежащими текущему пользователю
