1. Добавление удаления ключевых слов
UI изменения : В компоненте, где отображаются ключевые слова, добавьте иконку мусорки рядом с каждым ключевым словом.
Функционал : При нажатии на иконку мусорки отправляйте запрос на сервер для удаления этого ключевого слова из базы данных. Обновите UI после успешного удаления.
2. Добавление запроса на поиск в Perplexity API через n8n вебхук
Серверная часть : Создайте эндпоинт на сервере, который будет принимать список ключевых слов и отправлять их на n8n вебхук. После получения ответа сохраняйте найденные ссылки в таблицу campaign_links.
Клиентская часть : Добавьте кнопку или другой элемент управления для запуска поиска. После нажатия отправляйте запрос на серверный эндпоинт. Используйте таймер для обновления UI через 10 секунд после отправки запроса.
3. Добавление календаря постов и возможности добавить пост
UI изменения : Добавьте компонент календаря, где пользователь может видеть даты уже запланированных постов. Добавьте форму для создания нового поста (текст, текст с картинкой, видео).
Функционал : При создании нового поста отправляйте данные на сервер для сохранения в базе данных. Обновляйте календарь после успешного добавления поста.
Реализация функционала
4. Реализация функционала удаления ключевых слов
На стороне клиента : В компоненте, где отображаются ключевые слова, добавьте иконку мусорки. При нажатии на неё вызывайте функцию, которая отправляет DELETE-запрос на сервер с ID ключевого слова.
На стороне сервера : Создайте эндпоинт, который принимает DELETE-запрос и удаляет соответствующее ключевое слово из базы данных.
5. Реализация поиска ключевых слов и обновления UI
На стороне клиента : Добавьте кнопку "Поиск". При нажатии на неё отправляйте POST-запрос на сервер с массивом ключевых слов. После отправки запроса установите таймер на 10 секунд, по истечении которого обновите компонент, отображающий найденные ссылки.
На стороне сервера : Создайте эндпоинт, который принимает POST-запрос с массивом ключевых слов, отправляет их на n8n вебхук, получает ответ и сохраняет найденные ссылки в таблицу campaign_links.
6. Реализация календаря постов и добавления новых постов
На стороне клиента : Добавьте компонент календаря, который отображает даты запланированных постов. Добавьте форму для создания нового поста с полями для типа поста, контента, изображения и видео. При отправке формы отправляйте данные на сервер.
На стороне сервера : Создайте эндпоинт, который принимает данные нового поста и сохраняет их в таблицу posts. Также создайте эндпоинт для получения списка всех постов для конкретной кампании.
